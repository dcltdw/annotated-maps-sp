# A5.1c — Frontend Auth UI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the SPA a real login experience on top of the A5.1a/b backend: an auth client + bearer-token wiring, a topbar login/signup popover, `/auth/me` session restore on load, and a demo-login hint — while the anonymous `preview_as` persona switcher remains for logged-out visitors.

**Architecture:** A new `src/api/auth.ts` holds the auth calls + a `localStorage["authToken"]` store with a `getToken()` accessor. The existing `src/api/maps.ts` fetch wrappers attach `Authorization: Bearer <token>` whenever a token is present (and the write calls accept a nullable `previewAs`, so an authenticated user with no persona omits the query param and is identified by the bearer). A new `AuthBar` topbar component renders the login/signup popover (logged out) or a name + Log out chip (logged in). `MapScreen` holds the `authUser` state, restores it via `/auth/me` on load, hides the `PreviewSwitcher` when authenticated, and treats an authenticated user as able to write.

**Tech Stack:** React + TS + Vite, react-i18next, Vitest + Testing Library, Playwright (e2e).

## Global Constraints

- Token transport: attach `Authorization: Bearer <token>` from `localStorage["authToken"]` to API requests when a token exists; keep the existing `credentials: "include"` (the anonymous sandbox cookie-session still applies to logged-out visitors).
- Clear the stored token on logout and on a `401` from `/auth/me` (a stale/expired token). Do NOT auto-clear on a `401` from other endpoints.
- The `preview_as` persona switcher stays for logged-OUT visitors; it is hidden when a user is authenticated (a real user is not previewing).
- Demo-login hint copy must surface the public demo credential: `friend@demo.example` / `demo-pass-12345`.
- All user-facing strings go through i18n (`frontend/src/locales/en.json`) — no hardcoded copy in components (match the existing pattern).
- Gates after each task (REAL exit codes, run from `frontend/`, SEPARATELY — never `cmd | tail; echo $?`): `npm run test -- --run; echo $?` / `npm run lint; echo $?` / `npx tsc -b; echo $?` / `npm run build; echo $?`. The e2e task additionally runs `npm run e2e; echo $?`.

**Spec:** `docs/superpowers/specs/2026-06-21-a5.1-core-auth-design.md` §4 (frontend) + §7 (frontend testing). PR **A5.1c** (third of three: A5.1a foundation ✓ → A5.1b identity integration ✓ → **A5.1c frontend auth UI**). Work from `frontend/`. **UI decision (locked with the user):** the auth UI is a **topbar popover** — a compact "Log in" button that opens a small inline form; logged in shows a "name · Log out" chip; the persona switcher shows only when logged out.

**Backend contract (already shipped, A5.1a/b):**
- `POST /api/v1/auth/signup` `{email, password, display_name}` → `201 {token, user:{id,email,display_name,reputation}}` (dup email → 409, weak/blank → 422).
- `POST /api/v1/auth/login` `{email, password}` → `200 {token, user}` (bad creds → generic 401).
- `POST /api/v1/auth/logout` (bearer) → `204`.
- `GET /api/v1/auth/me` (bearer) → `200 {id,email,display_name,reputation}` or `401`.
- An authenticated bearer identity ALWAYS wins server-side; `preview_as` is ignored when a bearer is present. The read API's per-note `editable` is computed from the resolved identity, so the frontend just trusts `note.editable`.

**Current state (read before starting):**

- `src/api/maps.ts` has `getJson<T>(url)` and `sendJson<T>(url, method, body)` (both `credentials: "include"`); the write fns (`fetchNoteForEdit`, `createNote`, `updateNote`, `deleteNote`, `createAppend`, `updateAppend`) currently take `previewAs: string` and unconditionally append `?preview_as=`. `makeApiError`/`ApiError` live here.

```ts
import { API_BASE } from "./apiBase";
import { makeApiError } from "./maps";
import type { UserOut } from "./types";

const TOKEN_KEY = "authToken";

export function getToken(): string | null {
  try {
    return localStorage.getItem(TOKEN_KEY);
  } catch {
    return null; // localStorage unavailable (SSR/hardened browsers)
  }
}
export function setToken(token: string): void {
  try {
    localStorage.setItem(TOKEN_KEY, token);
  } catch {
    /* ignore */
  }
}
export function clearToken(): void {
  try {
    localStorage.removeItem(TOKEN_KEY);
  } catch {
    /* ignore */
  }
}

interface AuthResult {
  token: string;
  user: UserOut;
}

async function postAuth(path: string, body: unknown): Promise<AuthResult> {
  const res = await fetch(`${API_BASE}/auth${path}`, {
    method: "POST",
    credentials: "include",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  if (!res.ok) {
    let detail = `POST /auth${path} → ${res.status}`;
    try {
      const errBody = await res.json();
      if (errBody && typeof errBody.detail === "string") detail = errBody.detail;
    } catch {
      /* non-JSON body */
    }
    throw makeApiError(res.status, detail);
  }
  return (await res.json()) as AuthResult;
}

export async function login(email: string, password: string): Promise<UserOut> {
  const { token, user } = await postAuth("/login", { email, password });
  setToken(token);
  return user;
}

export async function signup(email: string, password: string, displayName: string): Promise<UserOut> {
  const { token, user } = await postAuth("/signup", {
    email,
    password,
    display_name: displayName,
  });
  setToken(token);
  return user;
}

export async function logout(): Promise<void> {
  const token = getToken();
  if (token) {
    try {
      await fetch(`${API_BASE}/auth/logout`, {
        method: "POST",
        credentials: "include",
        headers: { Authorization: `Bearer ${token}` },
      });
    } catch {
      /* best-effort; we clear locally regardless */
    }
  }
  clearToken();
}

export async function me(): Promise<UserOut | null> {
  const token = getToken();
  if (!token) return null; // no token → don't call /auth/me (it would 401)
  const res = await fetch(`${API_BASE}/auth/me`, {
    credentials: "include",
    headers: { Authorization: `Bearer ${token}` },
  });
  if (res.status === 401) {
    clearToken(); // stale/expired token
    return null;
  }
  if (!res.ok) throw makeApiError(res.status, `GET /auth/me → ${res.status}`);
  return (await res.json()) as UserOut;
}
```

```tsx
import { useState, type FormEvent } from "react";
import { useTranslation } from "react-i18next";
import { login, logout, signup } from "../api/auth";
import type { UserOut } from "../api/types";

interface Props {
  user: UserOut | null;
  onAuthed: (user: UserOut) => void;
  onLoggedOut: () => void;
}

export function AuthBar({ user, onAuthed, onLoggedOut }: Props) {
  const { t } = useTranslation();
  const [open, setOpen] = useState(false);
  const [isSignup, setIsSignup] = useState(false);
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [displayName, setDisplayName] = useState("");
  const [error, setError] = useState(false);
  const [busy, setBusy] = useState(false);

  if (user) {
    return (
      <div className="authbar">
        <span className="authbar__who">▸ {user.display_name}</span>
        <button type="button" onClick={async () => { await logout(); onLoggedOut(); }}>
          {t("auth.logout")}
        </button>
      </div>
    );
  }

  const submit = async (e: FormEvent) => {
    e.preventDefault();
    setBusy(true);
    setError(false);
    try {
      const u = isSignup ? await signup(email, password, displayName) : await login(email, password);
      setOpen(false);
      onAuthed(u);
    } catch {
      setError(true);
    } finally {
      setBusy(false);
    }
  };

  return (
    <div className="authbar">
      <button type="button" className="authbar__toggle" onClick={() => setOpen((o) => !o)}>
        {open ? t("auth.close") : t("auth.login")}
      </button>
      {open && (
        <form className="authbar__popover" onSubmit={submit}>
          {isSignup && (
            <label>
              {t("auth.displayName")}
              <input value={displayName} onChange={(e) => setDisplayName(e.target.value)} />
            </label>
          )}
          <label>
            {t("auth.email")}
            <input type="email" value={email} onChange={(e) => setEmail(e.target.value)} />
          </label>
          <label>
            {t("auth.password")}
            <input type="password" value={password} onChange={(e) => setPassword(e.target.value)} />
          </label>
          {error && <p className="authbar__error" role="alert">{t("auth.failed")}</p>}
          <button type="submit" disabled={busy}>{isSignup ? t("auth.signup") : t("auth.login")}</button>
          <button type="button" className="authbar__switch" onClick={() => { setIsSignup((s) => !s); setError(false); }}>
            {isSignup ? t("auth.haveAccount") : t("auth.needAccount")}
          </button>
          <p className="authbar__hint">{t("auth.demoHint")}</p>
        </form>
      )}
    </div>
  );
}
```
Note on accessible names (this avoids a real query-ambiguity): the popover **toggle** reads "Log in" when closed and "Close" when open, so once the form is open there is no second "Log in" button competing with the **submit** button (whose name is exactly "Log in" in login mode / "Sign up" in signup mode — matching `^log in$` / `^sign up$` in the tests). The "switch" button reads "Need an account? Sign up" / "Have an account? Log in" (distinct from the exact-match submit queries). The email/password `<input>`s bind to their `<label>` by nesting, which `getByLabelText` resolves.

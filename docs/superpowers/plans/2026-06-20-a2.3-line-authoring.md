# A2.3 — Region Notes: Line (Route) Authoring — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Let a user draw a **line/route** and save it as a `path`-anchored region note — reusing the A2.2b `ShapeDrawer` abstraction (the line draw mode + the shape→createNote flow already exist).

**Architecture:** Purely a MapScreen UI addition: parametrize the existing draw-entry by `DrawMode`, add a "Draw route" button that arms `drawMode="line"`, and make the region-edit "Redraw" use the note's current shape kind. The backend (`path` field + write API), the read-rendering (line layer, A2.1), `NoteEditor`'s shape anchor, and `createNote` sending `shape` are all already in place — no backend/editor/adapter change.

**Tech Stack:** React + TS; the `ShapeDrawer` port (terra-draw's `TerraDrawLineStringMode` already wired by the adapter).

**Spec:** `docs/superpowers/specs/2026-06-19-a2-region-notes-design.md` (§6 build sequence, line layer). PR **A2.3** (off main). A2.4 (circle) stacks on this.

**Conventions:** frontend from `frontend/`. Gates (REAL exit codes, run separately — never `cmd | tail; echo $?`): `npm run test -- --run; echo $?` / `npm run lint; echo $?` / `npx tsc -b; echo $?` / `npm run build; echo $?` / `npm run e2e; echo $?`.

Current draw entry (`MapScreen.tsx`): `handleStartDraw` hardcodes `setDrawMode("polygon")`; a "Draw area" button (`className="draw-area"`, shown when `canWrite && mode==="view" && !drawMode`) calls it; a "Redraw area" button (shown when editing a region) also calls it; a `drawing-hint` shows `t("screen.drawingHint")` while `drawMode` is set. `editorShape` is the `Shape | undefined` passed to `NoteEditor` (kind "polygon" | "line").
